## <a name="what-happens-when-a-package-is-installed"></a><span data-ttu-id="b954c-128">Co się dzieje, gdy pakiet jest zainstalowany</span><span class="sxs-lookup"><span data-stu-id="b954c-128">What happens when a package is installed</span></span>

<span data-ttu-id="b954c-129">Prostu, różnych narzędzi NuGet zwykle utworzyć odwołanie do pakietu w pliku projektu lub `packages.config`, następnie wykonaj operację przywracania pakietu, które skutecznie instaluje pakiet.</span><span class="sxs-lookup"><span data-stu-id="b954c-129">Simply said, the different NuGet tools typically create a reference to a package in the project file or `packages.config`, then perform a package restore, which effectively installs the package.</span></span> <span data-ttu-id="b954c-130">Wyjątek stanowi `nuget install`, która rozszerza tylko pakietu do `packages` folderu i inne pliki nie modyfikować.</span><span class="sxs-lookup"><span data-stu-id="b954c-130">The exception is `nuget install`, which only expands the package into a `packages` folder and does not modify any other files.</span></span>

<span data-ttu-id="b954c-131">Ogólny proces wygląda następująco:</span><span class="sxs-lookup"><span data-stu-id="b954c-131">The general process is as follows:</span></span>

1. <span data-ttu-id="b954c-132">(Wszystkie narzędzia, z wyjątkiem `nuget.exe`) rekord identyfikator pakietu i wersja pliku projektu lub `packages.config`.</span><span class="sxs-lookup"><span data-stu-id="b954c-132">(All tools except `nuget.exe`) Record the package identifer and version into the project file or `packages.config`.</span></span>

2. <span data-ttu-id="b954c-133">Pobieranie pakietu:</span><span class="sxs-lookup"><span data-stu-id="b954c-133">Acquire the package:</span></span>
   - <span data-ttu-id="b954c-134">Sprawdź, czy pakiet (przez dokładne identyfikatorem oraz numer wersji) jest już zainstalowany w *globalne pakiety* folderu zgodnie z opisem na [Zarządzanie globalne pakietów i foldery pamięci podręcznej](managing-the-global-packages-and-cache-folders.md).</span><span class="sxs-lookup"><span data-stu-id="b954c-134">Check if the package (by exact identifer and version number) is already installed in the *global-packages* folder as described on [Managing the global packages and cache folders](managing-the-global-packages-and-cache-folders.md).</span></span>

   - <span data-ttu-id="b954c-135">Jeśli pakiet nie znajduje się w *globalne pakiety* folderu, próbują pobrać go ze źródeł wymienionych na liście [pliki konfiguracji](Configuring-NuGet-Behavior.md).</span><span class="sxs-lookup"><span data-stu-id="b954c-135">If the package is not in the *global-packages* folder, attempt to retrieve it from the sources listed listed in the [configuration files](Configuring-NuGet-Behavior.md).</span></span> <span data-ttu-id="b954c-136">Dla źródeł online próbował najpierw pobrać pakiet z pamięci podręcznej, chyba że `-NoCache` zostanie określony z `nuget.exe` polecenia lub `--no-cache` zostanie określony z `dotnet restore`.</span><span class="sxs-lookup"><span data-stu-id="b954c-136">For online sources, attempt first to retrieve the package from the cache unless `-NoCache` is specified with `nuget.exe` commands or `--no-cache` is specified with `dotnet restore`.</span></span> <span data-ttu-id="b954c-137">(Visual Studio i `dotnet add package` zawsze używać pamięci podręcznej.) Jeśli pakiet jest używany z pamięci podręcznej, "Pamięci PODRĘCZNEJ" pojawia się w danych wyjściowych.</span><span class="sxs-lookup"><span data-stu-id="b954c-137">(Visual Studio and `dotnet add package` always use the cache.) If a package is used from the cache, "CACHE" appears in the output.</span></span> <span data-ttu-id="b954c-138">Pamięć podręczna zawiera czas wygaśnięcia 30 minut.</span><span class="sxs-lookup"><span data-stu-id="b954c-138">The cache has an expiration time of 30 minutes.</span></span>

   - <span data-ttu-id="b954c-139">Jeśli pakiet nie jest w pamięci podręcznej, próba go pobrać ze źródła w konfiguracji.</span><span class="sxs-lookup"><span data-stu-id="b954c-139">If the package is not in the cache, attempt to download it from the sources listed in the configuration.</span></span> <span data-ttu-id="b954c-140">Jeśli pakiet zostanie pobrany "GET" i "OK", są wyświetlane w danych wyjściowych.</span><span class="sxs-lookup"><span data-stu-id="b954c-140">If a package is downloaded, "GET" and "OK" appear in the output.</span></span>

   - <span data-ttu-id="b954c-141">Jeśli pakiet nie może zostać pomyślnie pobrany z żadnymi źródłami, instalacja nie powiedzie się w tym punkcie z powodu błędu takie jak [NU1103](../reference/errors-and-warnings.md#nu1103).</span><span class="sxs-lookup"><span data-stu-id="b954c-141">If the package cannot be successfully acquired from any sources, installation fails at this point with an error such as [NU1103](../reference/errors-and-warnings.md#nu1103).</span></span> <span data-ttu-id="b954c-142">Uwaga tego błędów z `nuget.exe` Pokaż polecenia zaznaczone tylko ostatni źródła, ale oznacza, że pakiet nie była dostępna z dowolnego źródła.</span><span class="sxs-lookup"><span data-stu-id="b954c-142">Note that errors from `nuget.exe` commands show only the last source checked, but implies that the package wasn't available from any source.</span></span>

   <span data-ttu-id="b954c-143">Podczas pobierania pakietu, mogą stosować kolejność źródeł w konfiguracji NuGet:</span><span class="sxs-lookup"><span data-stu-id="b954c-143">When acquiring the package, the order of sources in the NuGet configuration may apply:</span></span>

   - <span data-ttu-id="b954c-144">W przypadku projektów przy użyciu formatu PackageReference NuGet sprawdza źródła lokalnego folderu i udziałów sieciowych przed zaewidencjonowaniem źródła HTTP.</span><span class="sxs-lookup"><span data-stu-id="b954c-144">For projects using the PackageReference format, NuGet checks sources local folder and network shares before checking HTTP sources.</span></span>

   - <span data-ttu-id="b954c-145">Dla projektów przy użyciu `packages.config` format zarządzania, NuGet używa kolejność źródeł w konfiguracji.</span><span class="sxs-lookup"><span data-stu-id="b954c-145">For projects using the `packages.config` management format, NuGet uses the order of the sources in the configuration.</span></span> <span data-ttu-id="b954c-146">Wyjątek operacje przywracania, w którym to przypadku porządkowania źródła jest ignorowana i NuGet korzysta z pakietu z innego źródła odpowiada w pierwszej kolejności.</span><span class="sxs-lookup"><span data-stu-id="b954c-146">An exception is restore operations, in which case source ordering is ignored and NuGet uses the package from whichever source responds first.</span></span>

   - <span data-ttu-id="b954c-147">Ogólnie rzecz biorąc kolejność, w którym NuGet kontroli źródła nie jest szczególnie przydatne, ponieważ wszystkie danego pakietu o określoną liczbę identyfikator i wersja jest dokładnie taka sama niezależnie od źródła został znaleziony.</span><span class="sxs-lookup"><span data-stu-id="b954c-147">In general, the order in which NuGet checks sources isn't particularly meaningful, because any given package with a specific identifier and version number is exactly the same on whatever source it's found.</span></span>

3. <span data-ttu-id="b954c-148">(Wszystkie narzędzia, z wyjątkiem `nuget.exe`) Zapisz kopię pakietu oraz inne informacje w *pamięci podręcznej http* folderu zgodnie z opisem na [Zarządzanie globalne pakietów i foldery pamięci podręcznej](managing-the-global-packages-and-cache-folders.md).</span><span class="sxs-lookup"><span data-stu-id="b954c-148">(All tools except `nuget.exe`) Save a copy of the package and other information in the *http-cache* folder as described on [Managing the global packages and cache folders](managing-the-global-packages-and-cache-folders.md).</span></span>

4. <span data-ttu-id="b954c-149">Jeśli pobrano, zainstaluj pakiet w poszczególnych użytkowników *globalne pakiety* folderu.</span><span class="sxs-lookup"><span data-stu-id="b954c-149">If downloaded, install the package into the per-user *global-packages* folder.</span></span> <span data-ttu-id="b954c-150">NuGet tworzy podfolder dla każdego identyfikatora pakietu, a następnie tworzy oddzielne podfoldery dla każdej zainstalowanej wersji pakietu.</span><span class="sxs-lookup"><span data-stu-id="b954c-150">NuGet creates a subfolder for each package identifier, then creates subfolders for each installed version of the package.</span></span>

5. <span data-ttu-id="b954c-151">Aktualizacja innych projektów plików i folderów:</span><span class="sxs-lookup"><span data-stu-id="b954c-151">Update other project files and folders:</span></span>

    - <span data-ttu-id="b954c-152">Dla projektów przy użyciu PackageReference, zaktualizuj wykresu zależności pakietu, które są przechowywane w `obj/project.assets.json`.</span><span class="sxs-lookup"><span data-stu-id="b954c-152">For projects using PackageReference, update the package dependency graph stored in `obj/project.assets.json`.</span></span> <span data-ttu-id="b954c-153">Zawartość pakietu, same nie są kopiowane do folderu projektu.</span><span class="sxs-lookup"><span data-stu-id="b954c-153">Package contents themselves are not copied into any project folder.</span></span>
    - <span data-ttu-id="b954c-154">Dla projektów przy użyciu `packages.config`, skopiuj części pakietu rozwinięte zgodne platformy docelowej projektu do projektu `packages` folderu.</span><span class="sxs-lookup"><span data-stu-id="b954c-154">For projects using `packages.config`, copy those parts of the expanded package that match the project's target framework into project's `packages` folder.</span></span> <span data-ttu-id="b954c-155">(Przy użyciu `nuget install`, cały pakiet rozszerzonej jest kopiowana, ponieważ `nuget.exe` nie bada pliki projektu, aby zidentyfikować platformy docelowej.)</span><span class="sxs-lookup"><span data-stu-id="b954c-155">(When using `nuget install`, the entire expanded package is copied because `nuget.exe` does not examine project files to identify the target framework.)</span></span>
    - <span data-ttu-id="b954c-156">Aktualizacja `app.config` i/lub `web.config` Jeśli pakiet używa [źródła i konfiguracji pliku przekształcenia](../create-packages/source-and-config-file-transformations.md).</span><span class="sxs-lookup"><span data-stu-id="b954c-156">Update `app.config` and/or `web.config` if the package uses [source and config file transformations](../create-packages/source-and-config-file-transformations.md).</span></span>

6. <span data-ttu-id="b954c-157">Zainstaluj wszelkie zależności niższego poziomu Jeśli nie są jeszcze zainstalowane w projekcie.</span><span class="sxs-lookup"><span data-stu-id="b954c-157">Install any down-level dependencies if not already present in the project.</span></span> <span data-ttu-id="b954c-158">Ten proces może zaktualizować wersje pakietu w procesie, zgodnie z opisem w [rozpoznawania zależności](../consume-packages/dependency-resolution.md).</span><span class="sxs-lookup"><span data-stu-id="b954c-158">This process might update package versions in the process, as described in [Dependency Resolution](../consume-packages/dependency-resolution.md).</span></span>

7. <span data-ttu-id="b954c-159">(Tylko w programie visual Studio) Wyświetl plik readme pakietu, jeśli są dostępne w oknie programu Visual Studio.</span><span class="sxs-lookup"><span data-stu-id="b954c-159">(Visual Studio only) Display the package's readme file, if available, in a Visual Studio window.</span></span>
